# مشروع MediQueue 🏥

**MediQueue** هو نظام متكامل لإدارة العيادات الطبية، يهدف إلى تسهيل عملية حجز المواعيد وتنظيم طوابير الانتظار (Queue Management) للمرضى والأطباء بشكل فعال وسلس.

---

## 🚀 نظرة عامة على المشروع
النظام مبني باستخدام تقنية **ASP.NET Core MVC**، ويتبع نمط التصميم **N-Layer Architecture** (بشكل مبسط داخل المشروع) لضمان فصل المسؤوليات وسهولة الصيانة. يوفر النظام لوحات تحكم مخصصة لثلاثة أدوار أساسية:
1.  **المدير (Admin):** لإدارة العيادات، الأطباء، والمستخدمين.
2.  **الطبيب (Doctor):** لإدارة مواعيد العمل، رؤية الحجوزات، وإدارة طابور المرضى.
3.  **المريض (Patient):** للبحث عن العيادات، حجز المواعيد، ومتابعة دوره في الطابور.

---

## 🏗️ بنية المشروع (Project Structure)

يتبع المشروع الهيكلية التالية:

*   **Controllers/:** تحتوي على المنطق الخاص بمعالجة الطلبات (Requests) وتوجيه البيانات بين المستخدم والخدمات.
*   **Models/:** تضم الكيانات الأساسية لقاعدة البيانات (Entities) مثل `User`, `Clinic`, `Appointment`, و `Queue`.
*   **BL (Business Logic)/:** هي الطبقة الأهم حيث تحتوي على الخدمات (Services) التي تنفذ العمليات الحسابية والمنطقية (مثل `AppointmentService`).
*   **Views/:** تحتوي على واجهات المستخدم (Razor Views) مقسمة حسب كل ميزة.
*   **ViewModel/:** نماذج بيانات مخصصة لنقل البيانات بين المتحكم والواجهة دون كشف الكيانات الأصلية لقاعدة البيانات.
*   **Data/ (أو داخل Models):** تحتوي على `MediQueueContext` وهو المسؤول عن التواصل مع قاعدة البيانات عبر EF Core.
*   **wwwroot/:** الملفات الثابتة (CSS, JavaScript, الصور).

---

## 📂 أهم الملفات والمجلدات

### 1. `Program.cs`
هو نقطة انطلاق التطبيق. يتم فيه:
*   إعداد الخدمات (Dependency Injection).
*   إعداد قاعدة البيانات (Entity Framework).
*   إعداد نظام الهوية (Identity) لتأمين الحسابات.
*   تعريف مسارات التوجيه (Routing).

### 2. `Models/MediQueueContext.cs`
يمثل الجسر بين الكود وقاعدة البيانات. يحدد الجداول والعلاقات (Relationships) بين الكيانات مثل علاقة الطبيب بالعيادة والمريض بالمواعيد.

### 3. `BL (Business Logic)`
هذا المجلد يحتوي على الواجهات (Interfaces) وتطبيقاتها (Implementations):
*   `IAppointmentService`: لإدارة منطق الحجز.
*   `IQueueService`: لإدارة ترتيب المرضى في الطابور.
*   تساعد هذه الطبقة في جعل الكود قابلًا للاختبار (Unit Testing).

### 4. `Models/DbInitializer.cs`
مسؤول عن إنشاء البيانات الأولية (Seeding) مثل الأدوار (Roles) وحساب المدير الافتراضي عند تشغيل المشروع لأول مرة.

---

## 🔄 كيفية الربط وتدفق البيانات (Workflow)

يعتمد المشروع على مبدأ **Dependency Injection (DI)** للربط بين المكونات:
1.  عند طلب المستخدم لصفحة (مثلاً حجز موعد)، يتلقى الـ **Controller** الطلب.
2.  يطلب الـ **Controller** الخدمة المطلوبة من الـ **BL** (عبر الـ Interface).
3.  تقوم الخدمة بالتعامل مع الـ **Context** لجلب أو تعديل البيانات في قاعدة البيانات.
4.  يتم تحويل البيانات إلى **ViewModel** وإرسالها للـ **View** لتظهر للمستخدم.

## 🛠️ التقنيات المستخدمة
*   **Framework:** ASP.NET Core 9.0 (MVC)
*   **ORM:** Entity Framework Core
*   **Database:** SQL Server (مع دعم SQLite للتطوير المحلي)
*   **Identity:** Microsoft Identity for Auth & Roles
*   **Frontend:** Bootstrap, Razor Pages, JavaScript

---

## ⚙️ كيفية التشغيل
1.  تأكد من تثبيت **.NET SDK**.
2.  قم بتحديث سلسلة الاتصال (Connection String) في ملف `appsettings.json`.
3.  نفذ الأوامر التالية في التيرمينال:
    ```bash
    dotnet ef database update
    dotnet run
    ```
4.  يمكنك تسجيل الدخول كمدير باستخدام البيانات الافتراضية (يتم إنشاؤها عبر `DbInitializer`).
